# As-Tongji : 济济一堂

校园一体化信息平台

## 工程架构

采用前后端分离形式

前端使用vue+bs动态展示

后端使用模块化架构
一个采用gin+redis的服务器作为中转,该服务器只对关键数据+热点数据做持久化存储,其余数据会向其他模块获取并作缓存
其他模块为爬虫
目前的爬虫列表有
1块钱爬虫(node.js)
竞赛信息爬虫(python)

这些爬虫都可以通过http方式获取数据 边运行服务器边爬 在本地做持久化存储

## 项目结构

/Frontend 存放前端文件
/Backend 存放后端文件

## 信息表示

所有的数据皆视为一个信息
每个信息所携带的字段必须有:

- id:信息的唯一区分标志
- tags:信息的标签,可以为空
- data:信息源数据
- expire:过期时间,null为永不过期

例如用户本身也就是一个信息
如果要建立信息之间的关联
也是使用信息 相当于把信息当作一个指针
